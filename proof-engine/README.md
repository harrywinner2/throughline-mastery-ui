# proof-engine — an independent correctness proof

This is a second, deliberately independent implementation of Throughline's
geometry "angle relationships" engine, written in Python and sharing **no code**
with the canonical TypeScript engine (`web/src/engine/geometry.ts`). Its only
job is to prove that the product's correctness logic is right.

The argument for correctness here is over-determined rather than asserted.
First, `tests/test_invariants.py` proves the geometry from first principles with
property-based tests (Hypothesis): for randomly chosen inclinations it checks
that vertical angles are equal, linear pairs sum to 180°, corresponding /
alternate angles are equal and co-interior angles are supplementary **when the
lines are parallel**, and that the non-parallel "trap" figure genuinely breaks
the cross-line equalities. Second, `tests/test_agreement.py` loads 192 test
vectors that were generated by the TypeScript engine and asserts this Python
`solve()` reproduces every ground-truth value exactly. Two independent engines,
in two languages, agreeing across 192 generated cases plus property tests, means
the correctness that the LLM-facing product relies on lives entirely outside the
LLM.

`app.py` exposes the same engine as an optional FastAPI verifier service
(`GET /health`, `POST /verify`) so an external caller can double-check answers
against an engine implemented in a different language from the product.

## Running

```bash
pip install -r requirements.txt
pytest -q                          # invariants + the 192 agreement cases
uvicorn app:app --port 8000        # optional independent verifier service
```

Then, for example:

```bash
curl localhost:8000/health
curl -X POST localhost:8000/verify -H 'content-type: application/json' \
  -d '{"inclination":25,"parallel":true,"relationship":"co-interior",
       "givenAngle":3,"askAngle":5,"answerDeg":155}'
```
